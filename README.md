# 동적 프로그래밍
공부한 동적 프로그래밍</br>
동적 프로그래밍은, 문제가 같은 부분 문제, 그리고 최적부분 구조로 구성되어 있을 떄</br>
적용이 가능한 기법이다.</br>
여기서 같은 부분문제로 이루어져 있다는 점이 Devide & Conquer 패러다임의 문제와 유사할 수 있지만.</br>
D&C 문제는, 부분문제가 모두 독립된 문제라는것이 동적 프로그래밍 문제와의 차이다.</br>

최적 부분구조란 부분해를 전체 해를 구해내는데 사용될 수 있는 구조를 가짐을 의미한다.</br>
이 말은 달리 말하여 이전의 계산결과를 현재 계산에 활용할 수 있다는것을 의미한다.</br>
한마디로 부분 해를 재활용하여 전체 해를 계산하겠다는 의미이다.</br>

그렇기에 하향식 기법에서는 캐시를 통해 부분 해를 저장하고</br>
상향식 기법은 테이블을 생성해 부분해를 저장해가면서 목표해에 도달한다.</br>

동적 프로그래밍은 두가지 기법이 존재한다.
### 1. 하향식 (Memoization)
하향식 방식은, 가장 문제의 첫 상태를 정의하고 재귀적으로 솔루션 함수를 호출하면서</br>
DFS방식으로 상태 트리를 만들어가며 목표 해에 도달하는 방법이다.

### 2. 상항식 (Tabulation)
상향식 방식은, 수학적 귀납법과 비슷한 방식으로, 가장 기본적인 기저조건을 정의한 뒤,</br>
문제의 일반적인 규칙을 이해하여 만들어낸 일반 함수 Fx를 통해, 기저조건부터 목표 해까지</br>
모든 해를 구해내는 방식이다. 

### 대표적인 DP 알고리즘
Dijkstra : 구해진 부분 최단 거리에 그리디하게 최소 엣지를 선택 하여 다음 최단거리를 구함</br>
Bellman-ford : Dijkstra에서 그리디한 성질을 뺴고 모든 경로를 탐색하며 DP 테이블을 채워나가면서 최단거리를 구함</br>
